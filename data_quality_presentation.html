Methodology & Framework Design

Quality Dimension Selection Rationale

Selected Quality Dimensions

  1. Completeness
    • Measures missing values in critical fields
    • Essential for financial data integrity
  2. Validity
    • Validates data against business rules
    • Checks ranges, formats, and constraints
  3. Consistency
    • Ensures uniform formatting standards
    • Validates standardized codes and formats
  4. Uniqueness
    • Identifies duplicate records
    • Prevents data redundancy issues

Core Algorithm Implementation

def run_all_checks(self, df): # Execute quality checks df = self.check_completeness(df) df = self.check_consistency(df) df = self.check_validity(df) df = self.check_uniqueness(df) # Generate overall quality flag flag_columns = [col for col in df.columns if col.startswith('flag_')] df['flag_overall'] = df[flag_columns].max(axis=1) return df

Quality Metrics Calculation

def calculate_quality_score(self, df): total_records = len(df) issues_found = df['flag_overall'].sum() quality_score = ((total_records - issues_found) / total_records) * 100 return quality_score

Main Challenge & Technical Solution

Primary Technical Challenge

JSON Serialization Compatibility Issue

Problem: Pandas data types (Int64DType, Float64DType) cannot be directly serialized to JSON format, causing runtime errors during quality report generation.

Impact: Prevented automated generation of summary reports for quality metrics tracking.

Error: TypeError: Object of type Int64DType is not JSON serializable

Solution Approach

Root Cause Analysis: The DataFrame dtypes dictionary contains pandas-specific type objects that don't translate to JSON.

Solution: Convert all pandas data types to string representations before JSON serialization.

Technical Implementation

# Problem Code: report_data = { 'total_records': len(df), 'data_types': df.dtypes.to_dict() # Error here } # Solution: report_data = { 'total_records': len(df), 'data_types': {col: str(dtype) for col, dtype in df.dtypes.items()} }

Additional Challenges Addressed

  • Mixed Data Types: Financial columns with inconsistent formats
  • Missing Value Handling: Robust null value processing across all checks
  • Business Rule Validation: Complex regex patterns for format checking
  • Performance: Vectorized operations for efficient processing
